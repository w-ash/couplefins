### v0.9.0: Responsive Layout
**Goal**: Make the app usable on phones for quick checks between monthly reconciliation sessions.
**Context**: The current layout is desktop-only: fixed sidebar, wide transaction tables, no touch-optimized interactions. The couple primarily reconciles on a laptop, but wants to check settlement status, budget progress, and recent transactions on their phones.
**What this unlocks**: On-the-go access to settlement status, budget health, and transaction review.
**Key tech choices**: Sidebar becomes bottom nav below `md:` breakpoint. Transaction table switches to card layout on mobile. Touch targets sized for fingers.

#### App Shell

- [ ] **Responsive sidebar / bottom nav**
    - Effort: M
    - Dependencies: v0.8.0
    - What: Below `md:` breakpoint, sidebar collapses to a bottom navigation bar with icon-only tabs and labels. Identity switcher moves to a top bar or settings-accessible toggle. Changes to `AppLayout.tsx` and `Sidebar.tsx`.
    - Why: Foundation for all mobile page layouts

#### Page Layouts

- [ ] **Responsive page content**
    - Effort: M
    - Dependencies: "Responsive sidebar / bottom nav"
    - What: Transaction table: horizontal scroll or card layout per transaction on mobile. Budget page: progress bars stack to single column. Dashboard: stat grid becomes single column. MonthSelector: full-screen modal on mobile. Upload page: already mostly single-column, drag-drop zone needs touch support.
    - Why: Every page needs to render cleanly at small viewports

#### Touch Optimization

- [ ] **Touch-optimized editing**
    - Effort: S
    - Dependencies: "Responsive page content"
    - What: Larger tap targets on interactive elements. `inputMode="decimal"` on number inputs for numeric keyboard. Bigger checkbox hit areas in bulk split mode. Expanded row editor sized for touch.
    - Why: Editing on mobile requires finger-friendly controls

### v0.9.1: Transaction Exclusion
**Goal**: Flag individual transactions to exclude from reconciliation without deleting them.
**Context**: Sometimes a transaction is tagged `shared` in Monarch but shouldn't be included in reconciliation. Currently the only fix is editing the tags. A direct exclusion toggle is faster, especially on mobile.
**What this unlocks**: Quick "this one's not shared" corrections without editing tags or re-uploading.

#### Domain

- [ ] **Transaction exclusion flag**
    - Effort: S
    - Dependencies: v0.9.0
    - What: Add `is_excluded: bool = False` to `Transaction` entity. Alembic migration for the column. `reconcile()` in `src/domain/reconciliation.py` filters out excluded transactions.
    - Why: One-line domain change with high UX value

#### UI

- [ ] **Exclusion toggle in expanded row**
    - Effort: S
    - Dependencies: "Transaction exclusion flag"
    - What: "Exclude from reconciliation" toggle in the expanded transaction editor. Excluded transactions show grayed out / struck-through in the table. Count of excluded transactions shown in summary stats.
    - Why: Quick visual feedback and easy toggling
