# v0.8.x: Upload Polish & Data Quality

For strategic overview, see the [planning overview](README.md).

### v0.8.0: Upload Experience
**Goal**: Improve the upload workflow with drag-and-drop and upload history.
**Context**: The upload flow works correctly but uses a basic file input and provides no historical view of past uploads. The couple can't easily see when they last uploaded or what was in each upload.
**What this unlocks**: Faster uploads and visibility into upload history for debugging and audit.

#### Drag-and-Drop Upload

- [ ] **Drag-and-drop file input**
    - Effort: S
    - Dependencies: v0.7.1
    - What: Replace the plain `<input type="file">` on the Upload page with a drag-and-drop zone. Visual feedback on drag-over (border highlight, icon change). File name display after drop. Click-to-browse fallback. Purely frontend work in `UploadPage.tsx`.
    - Why: Standard UX expectation for file upload; reduces friction

#### Upload History

- [ ] **Upload history endpoint + UI**
    - Effort: M
    - Dependencies: v0.7.1
    - What: `GET /api/v1/uploads/history` — list all uploads with person name, upload date, filename, transaction count (new/updated). New section on Upload page showing past uploads as a table. Read-only (no rollback).
    - Why: "When did we last upload?" and "What was in that upload?" are common questions
    - Notes:
        - Uses existing `Upload` entity and `UploadRepository`
        - Needs a new repo method to join uploads with transaction counts
        - No rollback/delete functionality in this version — just visibility
