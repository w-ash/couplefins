### v0.4.0: Budget Tracking (Vertical Slice 1)
**Goal**: Set monthly budget limits per category group for shared expenses and track spending against them — both monthly and year-to-date.
**Context**: Beyond reconciliation, the couple wants to manage their shared spending targets. They review budgets together once a month and need both a current-month snapshot and a YTD cumulative view.
**What this unlocks**: Proactive budget awareness, not just retroactive reconciliation.
**Key tech choices**: CategoryGroupBudget entity with effective_from for versioning over time. Budgets are at the category group level, not individual category. YTD budget = monthly_amount × months elapsed.

#### Budget Management Epic

- [ ] **Budget CRUD use cases + API endpoints**
    - Effort: M
    - Status: Not Started
    - What: GET/POST/PUT/DELETE /api/v1/budgets — manage monthly budget limits per category group
    - Why: Backend for budget configuration
    - Dependencies: v0.2.x
    - Notes:
        - CategoryGroupBudget: group_id (FK to CategoryGroup), monthly_amount, effective_from (date)
        - Budgets reference CategoryGroup by ID, not free-text category name
        - effective_from allows changing budgets without losing history
        - GET returns both monthly and YTD data per category group:
            - **Monthly**: current month's spending vs. monthly_amount
            - **YTD**: cumulative spending (Jan–current month) vs. YTD budget (monthly_amount × months elapsed)
        - API accepts optional `year` param (defaults to current year)
        - Grand total row across all groups in both views

### v0.4.1: Month Finalization (Vertical Slice 2)
**Goal**: Lock completed months to prevent accidental changes after both people agree.
**Context**: Once reconciliation is agreed upon, the month should be locked. History browsing lives in the Dashboard (month navigation from v0.2.2), not a separate page.
**What this unlocks**: Confidence that past months won't shift unexpectedly.

#### Finalization Epic

- [ ] **Finalize month use case + API endpoint**
    - Effort: S
    - Status: Not Started
    - What: POST /api/v1/reconciliation/{year}/{month}/finalize — marks a month as finalized, prevents further uploads for that month
    - Why: Prevents accidental modification of agreed-upon reconciliation
    - Dependencies: v0.2.0
    - Notes:
        - Sets is_finalized=true, finalized_at=now on ReconciliationPeriod
        - Upload endpoint rejects CSVs for finalized months
        - Optional: allow unfinalizing with confirmation

- [ ] **Finalization UI on Transactions page**
    - Effort: S
    - Status: Not Started
    - What: "Finalize Month" button on the Transactions page (`/transactions`), visual indicator for finalized months, warning on attempt to upload to finalized month
    - Why: User-facing control for month locking — lives where the reconciliation data is reviewed
    - Dependencies: "Finalize month use case + API endpoint"
    - Notes:
        - Confirmation dialog (hard-to-reverse action): title restates consequence ("Finalize March 2026?"), body explains impact ("Uploads for this month will be blocked"), buttons "Cancel" (default focus) / "Finalize Month" (primary styling, not destructive — locking is protective, not destructive)
        - Finalized indicator: lock icon + text "Finalized" + muted timestamp — three channels, never icon alone
        - Unfinalizing (if supported): destructive confirmation dialog ("Reopen March 2026?"), "Reopen Month" button with destructive styling

- [ ] **Finalization indicators on Dashboard**
    - Effort: S
    - Status: Not Started
    - What: Show finalized status for each month in the Dashboard's month history list. Past month rows link to Transactions view for that month.
    - Why: At-a-glance history without a separate page
    - Dependencies: "Finalization UI on Transactions page", v0.2.2 "Dashboard page"
    - Notes:
        - Month history in Dashboard shows: month, total shared spending, net settlement, finalized status
        - Finalized status: lock icon + "Finalized" text label (icon alone is ambiguous for screen readers and colorblind users)
        - Click any month → navigate to `/transactions?year=YYYY&month=MM`
        - Filter by year
