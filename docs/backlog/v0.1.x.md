### v0.1.2: Design Foundations & Theme (Vertical Slice 3)
**Goal**: Implement the design system (fonts, colors, dark/light mode) so all subsequent UI work starts from a solid visual foundation.
**Context**: The design system is defined in `.claude/rules/web-design-system.md` (Satoshi font, warm neutrals, teal/coral) but nothing is applied — fonts aren't loaded, colors are raw Tailwind defaults, no dark mode exists.
**What this unlocks**: Every page built from here forward inherits correct typography, warm palette, and dark mode support.
**Key tech choices**: CSS custom properties via Tailwind v4 `@theme` for light/dark token switching. `@custom-variant dark` with class strategy. Synchronous `<script>` for FOIT prevention.

- [x] **Load Satoshi + Geist Mono fonts**

    - Why: Typography is the #1 signal of design intentionality. Default system fonts = AI slop.
    - Dependencies: v0.1.1
    - Notes:
        - `--font-sans: 'Satoshi', sans-serif` for all UI text
        - `--font-mono: 'Geist Mono', monospace` for IDs and technical identifiers only
        - Financial figures use `font-variant-numeric: tabular-nums` for alignment

#### Color System Epic

- [x] **Define semantic color tokens**
    - Effort: M
    - Status: Completed (2026-03-10)
    - What: Define warm neutral palette and semantic color tokens as CSS custom properties in `web/src/app.css`. Tokens swap between light and dark via `.dark` class on `<html>`.
    - Why: Semantic tokens (surface, on-surface, primary, muted) mean markup rarely needs `dark:` prefixes — colors swap automatically.
    - Dependencies: "Load Satoshi + Geist Mono fonts"
    - Notes:
        - Use `@theme` (NOT `@theme inline`) for all color tokens — `@theme inline` bakes values at build time, breaking dark mode
        - Light mode: warm off-white background, warm card surfaces, warm dark gray text
        - Dark mode: dark warm surfaces, light warm text
        - Semantic: teal for positive/income/under-budget, coral for negative/expense/over-budget
        - Set `color-scheme: light` on `:root` and `color-scheme: dark` on `.dark` for native browser elements
        - Follow shadcn naming convention: background/foreground, card/card-foreground, primary/primary-foreground, muted/muted-foreground

### v0.1.5: Use Case Architecture Refactor (Vertical Slice 6)
**Goal**: Refactor all 12 use cases to a uniform Command/Result/UseCase pattern for an auditable architecture.
**Context**: Use cases had inconsistent structure — some lacked Command/Result classes, UoW was injected via constructor, no transaction scoping via context manager, no attrs validators on Commands. The refactor establishes a rigid `execute(command, uow) -> Result` contract that makes every use case interchangeable at the infrastructure level.
**What this unlocks**: Uniform pattern for all future use cases. Every use case is locally comprehensible — read one file, understand the full flow.

#### Infrastructure Epic

- [x] **Add context manager protocol to UnitOfWork**
