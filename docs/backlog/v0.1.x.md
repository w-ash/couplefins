#### Data Model Epic

- [ ] **Define domain entities**
    - Effort: M
    - Status: Not Started
    - What: Immutable domain entities for Person, Transaction, Upload, CategoryGroup, CategoryMapping, CategoryGroupBudget, ReconciliationPeriod
    - Why: Core data structures that all business logic operates on
    - Dependencies: "Create Clean Architecture directory structure"
    - Notes:
        - Use frozen dataclasses or attrs
        - Transaction fields: date, merchant, category, account, original_statement, notes, amount, tags (list[str]), payer_person_id, payer_percentage (is_shared is a derived @property)
        - payer_person_id: who paid for this transaction (set from upload context)
        - payer_percentage: payer's share as int 0-100, default 50. Other person's share = 100 - payer_percentage. Input source is `sXX` tag in CSV, but the field is input-mechanism-agnostic
        - Person: id, name
        - Upload: id, person_id, filename, uploaded_at, period_year, period_month
        - CategoryGroup: id, name (e.g., "Food & Dining", "Home Expenses")
        - CategoryMapping: category (str, Monarch category name), group_id (FK to CategoryGroup)
        - CategoryGroupBudget: id, group_id (FK to CategoryGroup), monthly_amount, effective_from
        - ReconciliationPeriod: id, year, month, is_finalized, finalized_at, notes

#### CSV Parsing Epic

- [ ] **Implement Monarch CSV parser**
    - Effort: M
    - Status: Not Started
    - What: src/domain/ or src/application/ module that parses Monarch CSV into domain Transaction entities
    - Why: Core capability — everything depends on reading Monarch exports correctly
    - Dependencies: "Define domain entities"
    - Notes:
        - Columns: Date, Merchant, Category, Account, Original Statement, Notes, Amount, Tags
        - Parse Tags column (comma-separated) to detect "shared" tag and extract sXX split ratio
        - is_shared: derived property — true when payer_percentage is not None
        - payer_percentage: extract from sXX tag (e.g., s70 → 70), default 50; None for personal transactions
        - payer_person_id: set from the person_id parameter (the uploading person)
        - Amount: negative = expense, positive = income/refund
        - Pure function: CSV bytes/text + person_id → list[Transaction]
        - Note: Today the couple manually enters split percentages in their spreadsheet. In the future they'll use Monarch sXX tags. The parser handles the tag-based input; the internal model (payer_person_id + payer_percentage) is agnostic to the input source.
