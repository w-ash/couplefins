For strategic overview, see the [planning overview](README.md).

### v0.7.0: Spending Trend Charts
**Goal**: Visual spending trends per category group over time, on a new Insights page.
**Context**: The app has rich spending data across months but only shows it as tables and numbers. The couple wants to see trends — which categories are growing, how this month compares to average, and where the money is going.
**What this unlocks**: Data-driven spending conversations beyond "who owes whom this month."
**Key tech choices**: Recharts for charting (lightweight, React-native, works with Tailwind). New `/insights` page in sidebar. Backend reuses existing `compute_category_breakdowns()` run per-month across the year. All read-only — no new entities.

#### Backend

- [ ] **Spending trends API endpoint**
    - Effort: M
    - Dependencies: v0.6.1
    - What: New `GetSpendingTrendsUseCase` + `GET /api/v1/insights/spending-trends?year=2026`. Returns monthly spending per category group across a year. Also returns per-month totals and year total.
    - Why: Powers the chart data
    - Notes:
        - Reuses existing `compute_category_breakdowns()` from `src/domain/categories.py`
        - Runs it per-month across the year — with ~200 tx/month this is instant
        - Output: `list[MonthlyGroupSpending]` with year, month, group_id, group_name, amount

#### Frontend

- [ ] **Chart library setup**
    - Effort: S
    - Dependencies: none
    - What: Add Recharts dependency. Create `<SpendingTrendChart>` component with dark/light mode support via CSS custom properties from the design system.
    - Why: Charting foundation for this and future versions

- [ ] **Insights page**
    - Effort: M
    - Dependencies: "Spending trends API endpoint", "Chart library setup"
    - What: New `/insights` page added to sidebar (between Budget and Upload). Year selector. Stacked area or line chart showing monthly spending per category group. "This month vs. 3-month average" comparison cards per group. Category group filter toggles to show/hide series.
    - Why: Primary home for spending analysis

### v0.7.1: Budget + YoY Overlays
**Goal**: Overlay budget limits on spending charts and compare year-over-year.
**Context**: v0.7.0 shows raw spending trends. Adding budget lines shows where the couple intended to be. YoY comparison reveals whether spending is growing or shrinking.
**What this unlocks**: "Are we on track?" and "Are we spending more than last year?" in one view.

#### Budget Overlay

- [ ] **Budget lines on spending charts**
    - Effort: S
    - Dependencies: v0.7.0
    - What: When a category group has a budget, show a dashed horizontal line on the spending chart at the budget amount. Uses existing `resolve_effective_budget()` from `src/domain/budget.py` for each month (handles mid-year budget changes).
    - Why: Visual budget-vs-actual without switching to the Budget page

#### Year-over-Year

- [ ] **Year-over-year comparison**
    - Effort: M
    - Dependencies: v0.7.0
    - What: Extend `GetSpendingTrendsUseCase` to accept a comparison year. On Insights page: toggle to overlay previous year's data as a lighter/dashed series. Uses existing `get_shared_by_year()` repository method for each comparison year.
    - Why: Reveals spending growth/shrinkage and seasonal patterns
